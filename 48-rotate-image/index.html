48. Rotate Image

by

Duct Tape Programmer

  • Monday 11 Sep 2017
  • Comments
  • leetcode

Question

You are given an n x n 2D matrix representing an image.

Rotate the image by 90 degrees (clockwise).

Note:
You have to rotate the image in-place, which means you have to modify the input 2D matrix directly. DO NOT allocate another 2D matrix and do the rotation.

Example 1:

Given input matrix =
[
[1,2,3],
[4,5,6],
[7,8,9]
],

rotate the input matrix in-place such that it becomes:
[
[7,4,1],
[8,5,2],
[9,6,3]
]
Example 2:

Given input matrix =
[
[ 5, 1, 9,11],
[ 2, 4, 8,10],
[13, 3, 6, 7],
[15,14,12,16]
],

rotate the input matrix in-place such that it becomes:
[
[15,13, 2, 5],
[14, 3, 4, 1],
[12, 6, 8, 9],
[16, 7,10,11]
]

Solution

The problem can be solved easily if we remember a trick.

  • Clockwise rotation: Swap up-down and then swap elements zig-zag
  • Anticlockwise rotation: Swap left right and then swap elements zig-zag

For example for the given matrix
[
[1,2,3],
[4,5,6],
[7,8,9]
]

After swapping up-down we get
[
[7,8,9],
[4,5,6],
[1,2,3]
]
After swapping zig-zag we get
[
[7,4,1],
[8,5,2],
[9,6,3]
]

Swapping up-down is pretty easy and can easily be achieved by

for (int i = 0; i < matrix.length / 2; i++) {
    for (int j = 0; j < matrix[0].length; j++) {
        int temp = matrix[i][j];
        matrix[i][j] = matrix[matrix.length - i - 1][j];
        matrix[matrix.length - i - 1][j] = temp;
    }
}

Although, I had some trouble in doing zig-zag swap it required you to limit the swap. That is once we swap matrix[0][1] with matrix[1][0] we should not swap it again whent the value of i becomes 1 from 0. My initial approach was to limit this with a simple if check as shown below

for (int i = 0; i < matrix.length; i++) {
    for (int j = 0; j < matrix[0].length; j++) {
        if (i < j) {
            int temp = matrix[i][j];
            matrix[i][j] = matrix[j][i];
            matrix[j][i] = temp;
        }
    }
}

This made my solution pass the test but it did poorly in Accepted Solutions Runtime Distribution. After seeing some solutions in the Discussion I found out that j can start from i+1 since we alwasy skip swaps where i == j. I thought this will improve the performance in the Accepted Solutions Runtime Distribution since we are skipping one column although the time distribution was still the same.
