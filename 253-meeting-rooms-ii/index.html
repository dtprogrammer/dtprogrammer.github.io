253. Meeting Rooms II

by

Duct Tape Programmer

  • Saturday 23 Sep 2017
  • Comments
  • leetcode, leetcode-locked

Question

Given an array of meeting time intervals consisting of start and end times [[s1,e1],[s2,e2],...] (si < ei), find the minimum number of conference rooms required.

For example,

Given [[0, 30],[5, 10],[15, 20]],
return 2.

Solution

  • The idea to solve this problem is simple
  • First we need to sort the Interval on basis of their start time so that we can traverse them in the order they start.
  • We initialize our min heap (based on meeting end time) with the first interval and then traverse the remaining intervals to determine how many simultaneously running intervals we have.
    • If most early ending interval in the queue has a end time <= current interval start time we can reuse the room. So we just update the end time with the current interval end time.
    • Else we know that we need another room to accomodate this even so we push the current Interval on the min heap.

Time complexity

O (n log n)

Space complexity

Constant
