236. Lowest Common Ancestor of a Binary Tree

by

Duct Tape Programmer

  • Friday 15 Sep 2017
  • Comments
  • leetcode

Question

Given a binary tree, find the lowest common ancestor (LCA) of two given nodes in the tree.

According to the definition of LCA on Wikipedia: “The lowest common ancestor is defined between two nodes v and w as the lowest node in T that has both v and w as descendants (where we allow a node to be a descendant of itself).”

        _______3______
       /              \
    ___5__          ___1__
   /      \        /      \
   6      _2       0       8
         /  \
         7   4

For example, the lowest common ancestor (LCA) of nodes 5 and 1 is 3. Another example is LCA of nodes 5 and 4 is 5, since a node can be a descendant of itself according to the LCA definition.

Solution

This problem can be solved in different possible ways. EPI also has a solution for this which is pretty lengthy. The Discuss section on leetcode have a very simple solution which I learned and replicated.

There are two possibilities here:

  1. The nodes are in different subtree
  2. One of the nodes itself is the root.

With the above, we can write the solution below in which if the subtree contains both the nodes then the result is subtree root itself. If it contains either of the nodes then results is that node. If none of the above cause is true then it returns null.

Time complexity

The time complexity is O(n) since we traverse the tree to find the LCA.

Space complexity

The space complexity is constant although since the algorithm uses recursion there is O(log n = h) space required on the stack

Notes

  1. If left is null return right, if right is null return left, if both are null the root itself is returned.
